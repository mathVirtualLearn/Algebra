# Catálogo de componentes — Algebra

Inventario **vivo** de lo ya construido, para **reutilizar y no duplicar**.

> **Regla para los agentes (`algebra-ui`, `algebra-logic`)**
> 1. **Antes de crear** un componente, vista, caso de uso o mapper → **buscar aquí primero**. Si ya existe algo equivalente, reutilízalo.
> 2. **Al crear** algo nuevo y reutilizable → **añade una fila** en la tabla correspondiente (mismo PR/cambio).
> 3. Mantener las columnas al día (sobre todo `Props (primitivos)` y `Ubicación`).

Convención de "Props (primitivos)": lista de los **tipos primarios** que recibe (recuerda: las
vistas solo reciben primitivos + closures).

En **3×3 SÍ se detalla por método**: `threeDegenerateCramer` muestra Δ y cada Δᵢ EXPLÍCITOS por Sarrus (con los números = su valor) y concluye (Δ=0 y Δx=Δy=Δz=0 → indeterminado / Δ=0 y algún Δᵢ≠0 → incompatible); `threeDegenerateGauss` muestra la matriz escalonada y lee la fila nula (`0 = k` incompatible / `0 = 0` con libre). Solución única y métodos sin cambios; guarda divisiones por 0 eligiendo pivote/variable no nulos (incl. intercambio de filas en Gauss/RREF) | ✅ |
